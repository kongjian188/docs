## SPA3102 简介

这是一个有关如何配置SPA3102与FreeSwitch的简单配置，以便将呼入电话转发到FreeSwitch分机1001，并将呼入电话显示为FreeSwitch分机1000。任何与拨号号码不匹配的规则将通过SPA3102拨打（即不属于分机5000、9999、1000-1019等）。

## SPA3102 "SIP/2.0 401 Unauthorized" 问题

如果您的设备无法在FreeSWITCH上注册，请尝试将“高级”>“语音”>“线路1”>“SIP传输”从“UDP”更改为“TCP”。我相信这是SPA的一个BUG，Cisco 至少已经知道这个问题：[链接](https://supportforums.cisco.com/thread/2021647)。

## SPA3102 操作假设

您使用的是默认的SPA3102配置（出厂默认设置），具有启用了Web访问功能。

您已经安装了初始的FreeSwitch。

PSTN应答延迟: 0

如果您将此值设置为2或3（等待来电显示查询的时间足够长），任何包含逗号（或任何其他应转义字符）的来电显示名称都会导致FS向SPA发送400：Bad From: Header的错误返回，并且几乎无法看到任何结果，甚至在fs\_cli中也看不到，除非将sofia debug增加到最高级别。我通过将SPA设置为记录到syslog服务器并从日志中获得SPA的TP Parse Error: 32来发现了这个问题。迄今为止，我解决这个问题的唯一方法是在Linux上使用nfqueue。可在以下网址获得用于执行此操作的脚本和工具：<http://gitorious.org/spa3102-invite-packet-scrubber> \-- 2013年4月 -- 来自思科/Linksys/Sipura的最新固件升级（2012年2月）可能修复了此错误。

**传入PSTN来电显示**

如果您已经阅读并理解了关于SPA3102在任何带有逗号的PSTN呼叫中出错的警告，但仍希望将PSTN来电显示数据传递给FreeSWITCH，请将以下设置从默认值'no'更改为'yes'：

PSTN CID用于VoIP CID: yes

这个设置在上面的FXO定时器值所在的同一页上。请注意，您需要将SPA3102的“PSTN应答延迟”设置为3秒，以从PSTN获取CID数据。

#### PSTN呼叫者ID问题

如上所述，需要3秒的延迟，以便适配器可以获取呼叫者ID信息。

如果您仍然无法获取呼叫者ID信息，请检查是否将您的FXO端口连接到传真机或TAD的EXT端口。如果是这样，请获取一个-3dB的分频器，直接在源头处拆分电话线。传真机和TAD可以过滤出必要的CLID信息，因此SPA3102可能无法获取所需的信息。

**RTP数据包大小设置**

如果您使用连接到FXS端口的模拟电话进行呼出，并且被叫方听到声音不清晰，请按照以下操作：

**admin -> advanced -> Voice -> SIP**

RTP数据包大小：0.02

**DTMF RFC2833设置**

Cisco/Linksys对于RFC2833的实现称为AVT。为设置SPA-3102的RFC2833，请执行以下操作：

**admin -> advanced -> Voice -> Line 1**

DTMF Tx方法：AVT

**国际控制**

_**注意：**如果在外拨时遇到SIP 503错误（\[CS\_CONSUME\_MEDIA\] \[NORMAL\_TEMPORARY\_FAILURE\]），请在Voice->info选项卡中检查“Line Voltage”。我已经看到有报告显示它高达48V，但在我这个案例中，空闲时为22V，在使用时为6V-7V。我通过打电话然后再次检查信息页面来获得实际使用时的电压。它默认为30V，因此我猜它认为线路一直在使用中（尽管呼入仍然有效），所以会出现503错误。您可能需要或不需要更改这个值，但将其更改为10V修复了我的错误。_

线路使用电压：[空闲和使用时电压之间的值]

## 安全性

我将我的SPA3102设置为桥接模式，连接到一个拥有5000多人的公司LAN，并且我的服务器只接受低配置的PCI卡片，没有额外的以太网口，因此我在这里列出了一些安全措施以解释我的方式进行安全加固。

有3种加固的方式，使拨号用户必须输入PIN码、HTTP Digets和IP限制。第一种方法很容易设置，但遗憾的是，您需要等待3声蜂鸣声，输入PIN码，然后才能进行拨号。如果您设置了一个网关，可以设置HTTP Digest。30分钟后，我放弃了，但也许有人可以在某一天更新本操作说明。最后一个方法——按IP进行锁定——似乎最容易。

_**注意：** 我还没有测试过这种IP阻止的安全措施，但理论上应该可以正常工作。待我测试后，我会更新此页面_

## 替代方法

我围着圈子转来转去，试图解决这个问题 - 所以这部分是关于进入pstn呼叫的步骤，让其转到名为200的组 - 它还允许使用Line1模拟电话作为正常分机 - 在这种情况下是1019

  
**Freeswitch方面**  
在拨号计划/公共目录下创建一个新文件 - 我把它称为01-pstn-in.xml

把下面的内容放进去。。

```xml
<include>
  <extension name="pstn">
    <condition field="destination_number" expression="^(500)$">
      <action application="set" data="domain_name=$${domain}"/>
      <action application="transfer" data="200 XML default"/>
    </condition>
  </extension>
</include>
```

请注意，拨打的号码是500

创建directory/default/500.xml，内容如下

```xml
<include>
  <user id="500">
    <params>
      <param name="password" value="1234"/>
      <param name="dial-string" value="${sofia_contact(${dialed_user}@${dialed_domain})}"/>
    </params>
    <variables>
    </variables>
  </user>
</include>
```

这基本上是为了解决当3102尝试注册入站voip线路时出现的错误。

## PSTN到VoIP呼叫者ID错误

由于此设备已经过时，Linksys停止向其提供固件更新，最新固件似乎在设置PSTN到VoIP网关时在呼叫者ID名称字段中传递非法字符时存在问题。当将PSTN CID设置为VoIP CID后，如果PSTN提供商不发送任何呼叫者ID名称或发送错误的名称，设备似乎会将其传递给FreeSwitch而不将其放在""中，这导致FreeSwitch无法接受任何呼叫。为了解决这个问题，我想到了一个补丁，发布在[FS-7542](https://freeswitch.org/jira/browse/FS-7542)上。
