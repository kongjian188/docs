**当前行为:**

从ASTERSIK邀请到FS的INVITE:  
From: "+4711111111" &lt;sip:+4711111111@[xxx.yyyyyyy.no](http://xxx.yyyyyyy.no)\>;tag=as24269fa6  
Contact: &lt;sip:+4711111111@1.2.3.4>

P-Asserted-Identity: &lt;sip:+4712121212@[xyz.no](http://xyz.no)\>  
Privacy: none

  
这些信息通过Info应用程序在FS上收集。  
Caller-Username: \[+4712121212\] ;---- FS根据P-Asserted-Identity中的内容更改此字段  
Caller-Caller-ID-Name: \[+4711111111\]  
Caller-Caller-ID-Number: \[+4712121212\] ;---- FS根据P-Asserted-Identity中的内容更改此字段  
variable\_sip\_from\_user: \[+4711111111\]  
variable\_sip\_from\_uri: \[+4711111111@[xxx.yyyyyyy.no](http://xxx.yyyyyyy.no)\]  
variable\_sip\_contact\_user: \[+4711111111\]  
variable\_sip\_contact\_uri: \[+4711111111@1.2.3.4\]

然后FreeSWITCH移除P-Asserted-Identity头。

**我们希望有另一种行为：**
也许可以通过在SIP配置文件上设置某些内容来改变默认行为？
Alter_default_p_asserted_behavior=true

True: 可以从通道变量中读取P-Asserted-Identity，并且可以导出到Leg-B。Caller-Username和Caller-Caller-ID-Number不会被更改。

False: 与当前行为相同
